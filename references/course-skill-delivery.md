# 课程 Skill 交付规范

## 目标

让学员只靠课程讲义和 GitHub 素材包，就能安装课程依赖的 Skill 并复现成品。课程不能依赖老师电脑里未交付的隐藏环境。

## 一、识别课程依赖

通读课程路线、讲师手卡、学员讲义、提示词和实际演示记录，列出所有被调用的 Skill。逐个判断：

- **必须交付**：学员完成演示、作业或最终成品时需要调用。
- **只写安装入口**：官方插件、在线服务配套 Skill、授权不允许复制或体积不适合入库。
- **不进入课程包**：只用于做课、排版、飞书同步、质量检查，学员复现课程不需要。

不要按“本次对话加载过哪些 Skill”机械打包，必须按学员复现路径判断。

## 二、授权与来源

打包前确认每个 Skill 的所有权和许可证：

- 自有 Skill：可以整理后放入课程仓库。
- 第三方开源 Skill：确认许可证允许再分发，保留 `LICENSE`、版本和上游地址。
- 官方插件或授权不明 Skill：不复制缓存和源码，提供官方安装方式、最低版本和安装后检查方法。
- 无法确认授权：默认不上传，先向用户说明阻塞点。

## 三、便携化清理

课程包路径统一为：

```text
<课程目录>/素材包/skills/
├── README.md
├── install-course-skills.ps1
└── <skill-name>/
    ├── SKILL.md
    ├── agents/
    ├── scripts/
    ├── references/
    └── assets/
```

复制后检查并处理：

- 把 `C:\Users\某人`、`D:\某项目` 等个人绝对路径改成 `$HOME`、环境变量或相对路径。
- 模型和大缓存只写下载或环境变量配置，不上传模型本体。
- 排除 `.git`、`.env`、密钥、令牌、日志、临时文件、`__pycache__`、`.pyc`、`node_modules`、输出视频和原始素材。
- 不为了缩小体积删除运行必需的脚本、模板、规则或许可证。
- 第三方包增加上游说明，至少记录项目地址、版本和许可证。

## 四、安装入口

从 `assets/course-skill-package/` 复制一键安装脚本和 README 模板，再按课程实际 Skill 列表修改说明。

安装脚本必须：

- 默认安装到 `$HOME\.codex\skills`。
- 自动发现当前 `skills` 目录下包含 `SKILL.md` 的子目录。
- 默认跳过已有同名 Skill，不能静默覆盖。
- 显式使用 `-Force` 时先备份旧目录，再复制新版本。
- 安装完成后提示重新打开 Codex。

## 五、写入课件

至少同步以下位置：

1. `课程说明.md`：列出课程 Skill、用途、下载链接、一键安装命令和官方插件边界。
2. 使用该 Skill 的小节 `学员讲义.md`：在第一次实操前加入“课前准备”，写安装、重启和检查提示词。
3. 对应 `讲师手卡.md`：加入老师可以直接说出口的安装说明。
4. 对应 `学员讲义.xml`：与 Markdown 内容一致。
5. `素材包/提示词合集.md`：增加安装后检查提示词，学员可直接复制给 Codex。
6. 飞书课程父页面：提供统一的 Skill 下载入口。
7. 飞书相关小节：加入本节需要的 Skill、安装方式和检查方法。

凡是让学员调用 Skill 生成本地工具或文件的提示词，必须做到可独立复制执行：

- 明确写出要调用的 Skill 和能力。
- 给出输入文件绝对路径占位符，不把“刚才的文件”作为唯一定位方式。
- 写明输出目录、文件名和是否允许覆盖。
- 输入缺失、格式错误或候选文件不唯一时，要求 Codex 停止并报告，不得猜测。
- 写明验收标准，以及生成后是否需要直接打开或预览。

若飞书已被用户手改，只做局部更新；先读取当前区块，不能用本地全文覆盖。

## 六、GitHub 发布

- 默认使用课程资料私有仓库，在 `courses/<课程名>/素材包/skills/` 发布。
- 讲师手卡仍按课程规则决定是否上传，不因 Skill 打包而默认公开。
- 推送后验证远端 `main` 与本地提交一致。
- 把 GitHub 的具体 `skills` 目录链接回填到本地课程说明和飞书，不只链接仓库首页。

## 七、发布前验证

至少完成：

- 每个课程 Skill 都有 `SKILL.md`。
- PowerShell 安装脚本语法检查通过。
- 用隔离安装目录实际运行一次安装脚本，确认所有 Skill 被发现并复制。
- Python 脚本执行语法解析；项目模板做适合其技术栈的基础检查。
- 扫描个人绝对路径、密钥模式、缓存和大文件。
- 检查第三方许可证与上游说明。
- 检查 Markdown 与 XML 安装说明一致。
- Git 工作区干净，本地和远端提交一致。
- 回读飞书父页面和相关小节，确认标题、链接、命令和中文无乱码。

## 八、最终汇报

向用户明确提供：

- GitHub 课程 Skills 目录链接。
- 飞书课程主页和受影响小节链接。
- 本地 Skill 包 README 路径。
- 一键安装命令。
- 实际打包的 Skill 列表，以及只提供官方安装入口的依赖。
- 验证结果和 Git 提交号。
